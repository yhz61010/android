# /save-solution

把**当前会话中某个已解决的话题**的排障过程，提炼成一篇 Markdown 文档，保存到项目 `docs/` 目录。

用户传入的参数（如果有）：`$ARGUMENTS`

## 执行步骤

按以下顺序严格执行，**不要跳步**：

### 1. 回溯并界定话题范围

根据是否传入参数，分两种模式：

**模式 A —— 无参数（`$ARGUMENTS` 为空）**：
- 从当前消息向上回溯，找到**用户最近一次提出新问题 / 新报错 / 新需求**的那条消息，作为话题起点。
- 终点为当前消息（即用户调用本命令的位置）。

**模式 B —— 有参数（`$ARGUMENTS` 非空）**：
- 把参数当作话题概要（关键词或一句话描述），在**整个当前会话历史**中检索与之匹配的讨论段落。
- 起点：该话题最早被提出的那条消息；终点：该话题最后一次被讨论的那条消息。
- 如果会话中存在多个匹配段落，选择**与参数描述最贴合**的那一段；若仍无法判断，列出候选范围让用户选择，不要擅自合并。
- 如果在会话中**找不到任何匹配的话题**，停下来告诉用户："未在当前会话中找到与 `<参数>` 相关的讨论，请确认或换个关键词。" 不要硬编内容。

**两种模式共同的兜底**：
- 如果识别到的范围里找不到清晰的"问题 → 方案"闭环（例如只有讨论没有结论），停下来告诉用户："未能识别到完整的解决闭环，请告诉我从哪条消息开始总结。" 不要硬编内容。

### 2. 先汇报，后写文件（强制确认）

在写文件之前，**先向用户汇报**以下信息，等待用户确认：

- 识别到的话题范围：用一句话概括起点问题
- 拟用文件名：`docs/YYYY-MM-DD-<kebab-case-topic>.md`（日期取今天系统日期；topic 用英文短横线命名，准确描述话题）
- 拟包含的小节列表（背景 / 最终方案 / 注意事项 / 踩过的坑 / 参考，按实际有内容的列出）

用户确认后再进入下一步。如果用户要求调整范围或文件名，按用户意见修改后再次确认。

### 3. 生成文档内容

按下面的模板组织内容，使用 **UTF-8** 编码写入文件。

```markdown
# <话题标题>

> 日期：YYYY-MM-DD

## 背景 / 问题现象
<最初遇到的报错、需求或现象，尽量贴原始报错片段>

## 最终可行方案
<步骤化说明，含命令 / 代码块 / 文件路径 file:line>

## 注意事项
- <易错点 1>
- <易错点 2>

## 踩过的坑
1. **<失败尝试简述>** —— 为什么不行
2. ...

## 参考
- <相关源码路径 file:line>
- <外部链接，如果对话里出现过>
```

各小节生成规则：

| 小节 | 规则 |
|------|------|
| **背景** | 从用户最初提出问题的那条消息提取，保留关键报错原文，不要改写报错信息 |
| **最终方案** | 只写被用户确认"对了"的那一版；步骤化、可直接复制；命令保持原样 |
| **注意事项** | 提炼对话中的隐式前提（如"必须先 `source ~/.bashrc`"）、版本依赖、操作顺序要求 |
| **踩过的坑** | 列出此话题中尝试过但失败的方案；每条一句话说明失败原因，不展开细节 |
| **参考** | 涉及的代码位置使用 `path:line` 格式；外部链接仅收录对话中实际出现过的 |

**空小节处理**：若某节确实没有内容（例如完全没踩坑），**整节省略**而不是留空标题。

**语言约定**：
- 文件名：英文 kebab-case
- 正文：中文
- 代码块、命令、文件路径：保持原样不翻译

### 4. 写入文件（防覆盖）

- 目标路径：`docs/<filename>.md`（项目根的 `docs/` 目录，不是 `docs/superpowers/` 等子目录）
- **写入前必须检查同名文件是否存在**：
  - 不存在：直接写入
  - 已存在：自动追加后缀 `-2`、`-3`、… 直到不冲突，**禁止覆盖已有文件**
- 必须使用 UTF-8 编码

### 5. 输出结果

- 输出最终文件的**绝对路径**，方便用户打开查看
- 一句话说明文档包含哪些小节
- **不要执行 `git add` / `git commit` / `git status`**，本命令不涉及任何 git 操作

## 边界与禁止事项

- ❌ 不调用任何构建 / 测试 / lint 工具
- ❌ 不读取记忆系统（memory），只看当前对话上下文
- ❌ 不覆盖已有文件
- ❌ 不在对话内容不足以支撑结论时硬编方案
- ❌ 不执行任何 git 操作
- ✅ 只做"读对话 + 写一个 md 文件"这一件事
