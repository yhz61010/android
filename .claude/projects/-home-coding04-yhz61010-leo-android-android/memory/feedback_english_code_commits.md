name: English for code comments and commits
description: Always use English for code comments and git commit messages
type: feedback
---

Code comments and git commit messages must always be written in English.

**Why:** User's explicit requirement for consistency in code and version history.

**How to apply:** When writing or modifying code, use English for all comments. When creating git commits, write the commit message in English. This applies regardless of what language the user communicates in.
